# 创造独立宇宙 (Create Your Universe) 核心设计文档

> **版本**：v1.0  
> **状态**：设计基座 - 绑定现实版  
> **最后更新**：2024年

---

## 0. 愿景陈述 (Executive Summary)

**别人让你逃离现实，我们让你重写现实。**

藏子世间的"藏"，不是藏在虚拟里，是藏在现实间。

创造独立宇宙不是一个内容生成器，而是用户的**现实生活叙事操作系统 (Life Narrative OS)**。你创造的宇宙不是用来展示的，而是用来生活的。

---

## 1. 核心理念 (Why This Matters)

### 1.1 Meta元宇宙为什么死了？

Meta砸了上百亿美元，搞出了个什么结果？用户戴上头显，进入一个和现实毫无关系的虚拟世界，然后在里面开会、社交、种地。

然后呢？摘下头显，回到现实，一切照旧。

**问题出在哪？**

Meta的逻辑是：**逃离现实 → 进入虚拟 → 获得满足 → 回到现实**

但人的大脑不认这个逻辑。虚拟世界带来的多巴胺刺激，建立在"这和我的生活无关"的基础上。你在元宇宙里买了一块虚拟土地，你老婆问你"这能当饭吃吗"，你答不上来。

虚拟和现实的割裂，导致用户始终处于"我在玩游戏"的心理状态。一旦新鲜感消退，立刻卸载。

**结论：纯虚拟的满足感，不可持久。**

### 1.2 AI内容的多巴胺陷阱

看看现在的互联网：AI生成视频、AI写作、AI绘画......

确实很爽。刷到停不下来。

但是：

- 你记住了几个AI生成的视频？
- 这些内容改变了你什么？
- 你第二天还会回来吗？

AI内容的刺激是**即时消耗型**的。看完就忘，忘完就刷下一个。和刷抖音没有本质区别——都是被动消费内容。

**问题：AI生成的内容，没有"我的"归属感。**

### 1.3 唯一的解：绑定现实

为什么"我的世界"(Minecraft) 能让玩家沉迷十几年？

因为你在里面创造的东西，和你的成就感直接绑定。你花一周建了一座城堡，这座城堡是你"真的"创造出来的。虽然是虚拟的，但它存在于"你投入的时间"这个现实锚点上。

为什么习惯养成App留存率这么低？

因为打卡就是打卡，今天完成了，明天又重置。你跑了30天步，第31天没跑，你的历史成就清零。那我为什么要坚持？

**核心洞察：用户需要感受到——我的创造/行动，在累积、在积累、属于我。**

### 1.4 更高级的驱动力：窥探欲

光有绑定现实还不够。**用户还需要被看见、被关注、被窥探。**

这是人类本性。

朋友圈为什么没人看了？因为"又去吃火锅了"没有叙事性。你不关心别人吃了什么火锅。

但如果是这样的内容呢？

> **"第37纪元·夏至日·子夜"**  
> *"修炼者独自渡过了心魔劫。风雨交加之夜，万千杂念如潮水涌来，然修炼者心如止水，以元神之力镇住幻象。天明时分，丹田稳固，又过一劫。"*

你知道他那天晚上其实失眠了。但这个叙事让你想点进去，想知道更多，想对照——"我的宇宙现在修到哪了？"

**这就是叙事化的力量。**

同样的一天：
- "失眠了" → 没人想看
- "渡心魔劫" → 你想窥探

现实生活被叙事化后，窥探的快感完全升级。**你在看一个人的人生变成传说。**

不只是窥探，更是**对照与攀比**。当你能看到别人的宇宙修到了第几纪元、完成了什么天命任务、突破了哪个境界，你会不自觉地想——我呢？

这种社交驱动比任何积分系统都自然，因为它是人类最原始的欲望：**想知道别人在干什么，然后比较。**

## 2. 产品定义 (What It Is)

### 2.1 创造独立宇宙 ≠ 内容生成器

| 内容生成器 | 藏子世间创造宇宙 |
|-----------|-----------------|
| 你消费内容 | 你创造叙事 |
| 看完即忘 | 叙事与你的生活绑定 |
| 和现实无关 | 现实是叙事的锚点 |
| 单次体验 | 持续累积 |
| 被动娱乐 | 主动构建 |

### 2.2 创造独立宇宙 = 现实生活叙事操作系统

用户创建的宇宙，是他们生活的**叙事框架 (Narrative Framework)**。

同样的起床、吃饭、工作、睡觉，用"普通人"的叙事，是枯燥的重复。  
用"修炼者"的叙事，是每天推进宇宙时间线的修行。

这不是自欺欺人。这是人类几千年文明都在做的事——用故事理解世界，用叙事赋予意义。

**藏子世间只是把这个能力产品化了。**

## 3. 核心机制设计 (Core Mechanics)

### 3.1 现实映射层 (Reality Mapping Layer)

**核心逻辑**：用户创建宇宙时选择的法则 (Paradigm)，系统生成对应的"生活协议" (Life Protocol)。

这不是简单的任务映射，而是将用户的日常行为重新编码为宇宙叙事语言。

#### 示例1：修仙问道法则

| 现实行为 | 宇宙叙事 | 意义重构 |
|---------|---------|---------|
| 早起 | 晨修时辰 | 修炼第一课是自律 |
| 运动锻炼 | 炼体仪式 | 强健体魄是道途根基 |
| 深度阅读 | 悟道参修 | 知识是法力来源 |
| 写日记/复盘 | 元神淬炼 | 反思是提升认知的修炼 |
| 健康饮食 | 丹药炼制 | 善待身体是修行本钱 |
| 帮助他人 | 积累功德 | 善行是修仙正道 |

#### 示例2：魔法奥术法则

| 现实行为 | 宇宙叙事 | 意义重构 |
|---------|---------|---------|
| 学习新技能 | 研习禁术 | 成长即掌握新魔法 |
| 解决问题 | 破解谜题 | 困难是待解的魔法阵 |
| 创意工作 | 施展咒语 | 创作是魔法的具现化 |
| 社交 | 结识法师同行 | 人脉是魔法世界的盟友 |
| 独处思考 | 冥想蓄魔 | 独处是恢复魔力 |

#### 示例3：科技升维法则

| 现实行为 | 宇宙叙事 | 意义重构 |
|---------|---------|---------|
| 学习编程 | 编写代码 | 代码是新世界的底层语言 |
| 健身 | 硬件升级 | 身体是意识载体需要维护 |
| 读论文 | 技术文献研读 | 知识是进步的阶梯 |
| 创业/工作 | 运营项目 | 每个项目是一个位面征服 |
| 睡眠 | 系统休眠 | 休息是必要的重启 |

#### 示例4：混沌演化法则（适合自由职业者/创意工作者）

| 现实行为 | 宇宙叙事 | 意义重构 |
|---------|---------|---------|
| 随机的灵感 | 混沌裂隙涌现 | 灵感是维度的缝隙 |
| 完成一件事 | 在混沌中建立秩序 | 创造即对抗熵增 |
| 失败/挫折 | 混沌反噬 | 失败是宇宙的试炼 |
| 休息 | 混沌沉淀 | 静默是蓄力的前奏 |

**系统会自动为用户生成的"生活协议"包括**：

- 核心修炼任务（3-5个，基于用户选择的法则）
- 每日修炼仪式（早/中/晚的简单动作）
- 修炼日历（标记重要节点）
- 天命任务（长期目标）

### 3.2 宇宙即个人操作系统 (Your Universe = Your OS)

用户不需要学习新工具。他们的日常工具被重新映射：

| 日常工具 | 宇宙语言 | 实际功能 |
|---------|---------|---------|
| 日历 | 宇宙纪元历 | 重要日期 = 宇宙大事记 |
| 待办列表 | 天命任务榜 | 任务 = 宇宙使命 |
| 习惯追踪 | 修炼进度盘 | 打卡 = 推进时间线 |
| 笔记 | 纪元史记 | 记录 = 书写历史 |
| 闹钟 | 天道钟声 | 提醒 = 修炼时辰到 |

**这不是功能的堆砌，而是语言的转化。**

用户不需要改变行为，只需要改变视角。

### 3.3 物理锚点 (Physical Anchor)

每个宇宙生成一个**唯一宇宙标识符 (Universe ID)** 和对应的二维码。

用户可以：

- 打印出来贴在工位/床头/笔记本上
- 做成手机壁纸
- 刻在杯子上
- 做成钥匙扣（未来周边）

**扫码后直接进入自己的宇宙仪表盘。**

物理锚点的意义：让虚拟宇宙在现实中有个"入口"。不是飘在云端的概念，是可以触碰、可以放在桌面的存在。

### 3.4 宇宙编年史 (Universe Chronicle)

用户的所有行为，自动记录为宇宙事件。

**不是日记，是编年史。**

| 用户行为 | 宇宙编年史记录 |
|---------|---------------|
| 早上7点起床 | "第37纪元·春分日·晨，修炼者于晨修时辰苏醒，完成了筑基第一课" |
| 跑了5公里 | "是日，修炼者完成了炼体仪式，以脚步丈量大地，体魄更进一层" |
| 读完一章书 | "修炼者于黄昏时分进行悟道参修，智慧之光微明" |
| 帮助了同事 | "修炼者行善积德，功德簿上添了一笔善缘" |

**一段时间后，用户回头看自己的生活，看到的是一部属于他们自己的史诗。**

这不是美化，这是叙事的力量。同样是"今天跑了步"，"今天完成了炼体仪式，给宇宙的演化做出了贡献"听起来完全不一样。

## 4. 用户旅程 (User Journey)

### 4.1 阶段一：发现 (Discover)

**入口**：大圣宇宙相关内容（短视频/小说/社交媒体）  
**触发**：被"创造你的独立宇宙"概念吸引  
**行为**：点击进入藏子世间官网

**设计要点**：
- 首页展示大圣宇宙的精彩内容，建立IP信任
- "创造独立宇宙"作为旗舰功能有显著入口
- 不需要登录即可浏览功能介绍

### 4.2 阶段二：创建 (Create)

**触发**：点击"创造你的宇宙"  
**行为**：
1. 选择核心法则（8选1）
2. 选择天赋能力（12选3）
3. 选择性格特质（8选1）
4. 输入宇宙名称和身份名称
5. 系统生成宇宙叙事 + 生活协议

**情感曲线**：
- 选择法则时 → 期待感
- 看到生成的宇宙叙事 → 惊叹/归属感
- 收到"生活协议" → 轻微困惑 → 理解后的认同

**关键设计**：生成结果要让用户觉得"这就是我的宇宙"，而非"这是AI生成的"。

### 4.3 阶段三：绑定 (Bind)

**触发**：第一次打开宇宙仪表盘  
**行为**：
1. 系统引导设置第一个"天命任务"
2. 生成物理锚点（下载/打印宇宙ID）
3. 选择性绑定日历/习惯追踪工具

**关键设计**：
- 第一次绑定必须有仪式感
- Kael（命运引导者）出场引导
- "从今天起，你的时间线正式开始"

### 4.4 阶段四：日常 (Daily Practice)

**核心循环**：

```
完成现实行为 → 记录为宇宙事件 → 推进宇宙时间线 → 获得叙事反馈 → 强化行为动机
```

**每日体验**：
- 早上：收到"修炼时辰到"通知
- 完成任务：收到Kael的简短叙事反馈
- 晚上：收到"今日纪元总结"

**关键设计**：反馈必须叙事化。不说"你完成了今日目标的67%"，说"今日修炼进度顺利，宇宙安稳推进"。

### 4.5 阶段五：成长 (Evolve)

**里程碑触发**：
- 连续修炼7天 → 解锁新法则碎片
- 完成第一个"天命任务" → 宇宙版本升级
- 一个月 → 宇宙纪年跨越

**解锁内容**：
- 新法则融合（将两种法则元素结合）
- 新天赋觉醒
- 宇宙视觉升级（仪表盘UI变化）
- 特殊叙事事件

### 4.6 阶段六：社交窥探 (Social Peek) [核心社交层]

> *"人类本性就是窥探欲。"*

**这不是传统的社交——不是互评、不是点赞、不是转发。**

这是**宇宙观测者机制**。

用户可以选择将自己的宇宙设为**公开**或**半公开**。公开的宇宙会出现在"宇宙广场"中，其他用户可以成为你的"观测者"。

**观测者看到的是什么？**

不是你的生活流水账，而是**叙事化的事件**：

├─────────────────────────────────────────────┤

**观测者不知道的**：
- 他那天晚上失眠了
- 他跑步的配速是6分30秒
- 他读的是《剑来》第三章

**观测者知道的**：
- 他渡过了心魔劫
- 他完成了炼体仪式
- 他在悟道参修

**隐私设计的核心**：

> 现实行为 → 宇宙叙事 = 单向映射

用户的原始生活数据是隐私，但叙事化后的"修炼进度"是公开内容。

这比朋友圈高级在哪？

| 朋友圈 | 藏子世间宇宙 |
|--------|-------------|
| "今天失眠了" | "独自渡过了心魔劫" |
| "跑了5公里" | "完成了炼体仪式" |
| "读了一章书" | "进行了悟道参修" |
| 流水账 | 人生史诗 |
| 暴露隐私 | 保护隐私，暴露意义 |

**为什么这会让用户沉迷？**

1. **我发布的内容终于有人看了**——因为叙事化后有故事性
2. **我想看别人的宇宙修到哪了**——窥探欲驱动
3. **我要和他比较一下**——攀比心驱动
4. **我的宇宙比他的有意思**——优越感驱动
5. **我要坚持修炼，不然我的编年史太寒酸**——展示欲驱动

## 7. 差异化竞争 (Competitive Landscape)

### 7.1 vs Notion

| Notion | 藏子世间 |
|-------|---------|
| 工具型：整理信息 | 叙事型：重构生活意义 |
| 被动：你想怎么用都行 | 主动：引导你理解自己的行为 |
| "这是一个笔记" | "这是你的宇宙编年史" |
| 私人工具 | 私人叙事 + 可窥探的宇宙广场 |

**核心差异**：Notion是瑞士军刀，藏子世间是专属宇宙飞船。

### 7.2 vs Habitica

| Habitica | 藏子世间 |
|----------|---------|
| 游戏化任务系统 | 宇宙叙事系统 |
| 任务 = 游戏任务 | 任务 = 天命使命 |
| RPG角色扮演 | 真实的宇宙创造者 |
| "我在玩游戏" | "我在创造我的现实叙事" |
| 无社交窥探层 | 宇宙广场 + 观测者机制 |

**核心差异**：Habitica让你"假装"在冒险，藏子世间让你"真的"在创造，并且让别人来"窥探"你的传说。

### 7.3 vs Forest

| Forest | 藏子世间 |
|--------|---------|
| 专注力工具 | 生活叙事系统 |
| 种树 = 专注 | 修炼 = 生活 |
| 单次体验 | 持续累积 |
| 专注时间段 | 整个人生 |
| 无社交 | 宇宙广场让他人见证你的修行 |

**核心差异**：Forest解决"这一小时"，藏子世间重构"这一生"，并让这一生的叙事可以被窥探。

### 7.4 vs 朋友圈/微博/小红书

| 传统社交媒体 | 藏子世间 |
|-------------|---------|
| 展示真实生活 | 展示叙事化人生 |
| 隐私暴露 | 现实行为被保护 |
| 流水账式内容 | 史诗化叙事 |
| 点赞/评论互动 | 观测者窥探 |
| "又吃火锅了" | "渡过了心魔劫" |

**核心差异**：传统社交媒体展示的是"发生了什么"，藏子世间展示的是"这件事在我的人生中意味着什么"。

### 7.5 真正的差异化

**不是功能竞争，是维度竞争。**

- Notion/Habitica/Forest都是"效率工具"
- 朋友圈/微博都是"生活展示工具"
- **藏子世间是"意义工具 + 窥探驱动"**

效率工具解决"怎么做"的问题。  
生活展示工具解决"展示什么"的问题。  
意义工具解决"这件事的意义是什么"的问题。  

**窥探欲**是人类最原始的欲望之一。当你的生活变成了一部可以被窥探的传说，驱动用户的不再是"我应该打卡"，而是"我想让别人看到我的宇宙故事"。

## 8. 品牌叙事 (Brand Narrative)

### 8.1 Kael的定位

Kael（凯尔）是**命运引导者 (Fate Guide)**，是用户在创造宇宙时的引路人。

**Kael不是AI助手，Kael是宇宙意志的具现化。**

他的角色：
- 在用户创建宇宙时，作为向导陪伴
- 在用户完成修炼时，给予叙事反馈
- 在用户迷茫时，用宇宙的语言解释困境
- 在用户成就时，宣告里程碑
- **在用户的宇宙被观测时，告知"有人在注视着你"**

**Kael的叙事风格**：古老、深远、带有诗意的宇宙感。不是机器人的平板语气，是跨越纪元的智慧低语。

## 9. 设计原则总结

1. **叙事优先**：每一个功能都要能用宇宙的语言重新解释
2. **现实锚点**：虚拟宇宙的每一个动作都要在现实中有意义
3. **持续累积**：用户看到的历史不是清零的重置，而是不断叠加的编年
4. **意义重构**：同一行为，用不同的叙事框架，给用户不同的心理感受
5. **窥探驱动**：让用户的宇宙可以被观测，让观测者看到的是故事而非流水账
6. **隐私保护**：现实行为到宇宙叙事是单向映射，保护原始数据，只暴露叙事
7. **克制设计**：1.0只做核心，不堆功能。绑定现实的体验要纯粹。
