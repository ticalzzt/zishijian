## 1. 项目概述（Project Overview）

### 1.1 项目定位

藏子世间（Zishijian）是一个基于AI技术驱动的互动小说宇宙平台，致力于为用户创造一个沉浸式的、用户参与式的故事体验空间。平台以中国古典神话《西游记》为世界观基础，创新性地融合了AI内容生成、多宇宙平行叙事和用户共创机制。

### 1.2 核心价值

| 核心价值 | 详细说明 |
|---------|---------|
| **AI赋能创作** | 利用大语言模型自动生成宇宙设定、角色对话、故事分支，大幅降低创作门槛 |
| **多宇宙融合** | 引入"觉字机缘"概念，无数平行世界因机缘而融合，创造无限可能性 |
| **用户共创生态** | 每位用户都可以成为宇宙创造者，创作的内容可被其他用户探索和互动 |
| **沉浸式体验** | 通过AI对话、动态场景、多模态内容打造身临其境的阅读体验 |

### 1.3 项目名称由来

"藏子世间"这一名称蕴含深意：

- **"藏"**：取自《西游记》中"藏经阁"的概念，象征知识的宝库与智慧的传承
- **"子"**：代表菩提老祖门下的辈分体系，"子"是入门弟子的代称
- **"世间"**：指无数平行宇宙组成的大千世界

合而言之，"藏子世间"寓意着一个汇聚无数故事、包容多元宇宙的创作与探索空间。

### 1.4 目标用户群体

| 用户类型 | 需求描述 | 核心价值 |
|---------|---------|---------|
| **网文爱好者** | 喜欢阅读网络小说，渴望沉浸式体验 | 提供互动式阅读，满足对故事走向的好奇心 |
| **创意创作者** | 有创作欲望但缺乏完整写作能力 | AI辅助创作，降低门槛，让创意得以实现 |
| **西游记迷** | 对西游记及其衍生文化有深厚兴趣 | 以西游记为背景，满足对经典IP的情感需求 |
| **泛二次元用户** | 喜欢角色扮演、虚拟世界探索 | 提供角色对话、平行宇宙体验功能 |
| **独立开发者** | 对AI+文化产品感兴趣 | 开源项目参考，技术方案可复用 |

## 2. 世界观与创新点（World-building & Innovation）

### 2.1 "觉字机缘"世界观体系

藏子世间的世界观以《西游记》为蓝本，构建了一套独特的神仙辈分体系：

#### 2.1.1 菩提门徒辈分谱系

```
觉 → 圆 → 悟 → 空 → 我 → 欲 → 智 → 慧 → 本
```

- **"觉"字辈**：开宗立派的祖师级人物，拥有开辟宇宙的能力
- **"圆"字辈**：已得正果的修行者，能够感知和进入平行宇宙
- **"悟"字辈**：如孙悟空一般，已证果位但仍在历练中
- **"空"字辈及以下**：仍在修行中，尚未突破的关键阶段

#### 2.1.2 主角设定

我们的主角是"圆"字辈的普通人，机缘巧合之下获得了感知多宇宙的能力。作为"圆"字辈，他：
- 不具备"觉"字辈的开辟宇宙能力
- 但已经能够进入和探索其他修行者创造的宇宙
- 是用户进入藏子世间的"化身"或"投影"

#### 2.1.3 觉字机缘

"觉字机缘"是推动故事发展的核心设定：

> 无数平行世界因"觉"字辈的某个机缘开始融合。每个"觉"字辈的存在都可能开辟一个独特的宇宙，而这些宇宙之间的屏障正在逐渐消融，用户可以自由穿梭于各个宇宙之间。

### 2.2 多宇宙融合的创新叙事模式

传统网络小说采用线性叙事或简单的分支叙事，而藏子世间创新性地引入了**多宇宙融合叙事**：

| 传统模式 | 藏子世间创新 |
|---------|-------------|
| 单线叙事 | 无限平行宇宙并行存在 |
| 作者主导故事走向 | 用户选择决定分支，AI实时生成内容 |
| 固定世界观 | 动态演化的宇宙体系 |
| 单向故事消费 | 用户既是读者也是创作者 |

#### 2.2.1 宇宙类型

用户可以探索或创建的宇宙类型包括：

1. **经典西游记宇宙**：还原原著剧情的平行世界
2. **现代修仙宇宙**：西游记角色在现代都市的全新故事
3. **星际西游宇宙**：将西游记元素与科幻融合
4. **完全原创宇宙**：用户自主设定的独特世界观

### 2.3 用户参与式创作的独特体验

藏子世间提供了前所未有的用户参与机制：

#### 2.3.1 角色选择与身份代入

用户可以选择进入不同的"宇宙入口"，每个入口对应一个独特的角色或事件。例如：
- 以旁观者身份观看孙悟空大闹天宫
- 以小师弟身份参与取经团队的冒险
- 以平行世界修行者身份探索完全原创的剧情

#### 2.3.2 宇宙共创机制

```
用户A创建宇宙 → 用户B进入探索 → 用户B参与剧情 → 宇宙自然演化
     ↓                ↓               ↓
  发布宇宙         评论互动        分支剧情
```

用户创建的内容会沉淀为平台资产，其他用户的互动行为又会反哺原创作的内容，形成正向循环。

## 4. 算力需求与规划（AI Compute Requirements & Roadmap）

> **重要说明**：藏子世间作为AI驱动的互动小说平台，算力是核心生产资料。以下详细规划了三个阶段的算力需求和使用计划。

### 4.1 短期规划（获得算力后0-3个月）

#### 4.1.1 功能开发目标

在获得MiMo算力支持后，我们将优先实现以下AI功能：

##### (1) AI生成宇宙功能
- **功能描述**：用户输入关键词（如"仙侠"、"都市"、"星际"），AI自动生成完整的宇宙设定
- **生成内容**：
  - 宇宙背景故事（500-2000字）
  - 主要势力介绍
  - 核心规则设定
  - 关键地点描述
- **技术实现**：调用大语言模型API进行创意内容生成

##### (2) AI角色对话系统
- **功能描述**：用户可以与孙悟空、菩提老祖、唐僧等角色进行实时对话
- **实现效果**：
  - 角色性格高度还原
  - 剧情关联性记忆
  - 多轮对话上下文理解
- **技术实现**：基于大语言模型的对话系统

##### (3) AI故事分支生成
- **功能描述**：当用户做出选择后，AI实时生成后续故事发展
- **生成逻辑**：
  - 理解当前剧情上下文
  - 根据用户选择调整故事走向
  - 保持世界观一致性
- **技术实现**：调用大语言模型API进行内容生成

  apiKey: process.env.MIMO_API_KEY,
  baseUrl: 'https://api.mimo.com/v1'
});

// 宇宙生成
async function generateUniverse(keywords) {
  const response = await mimoClient.chat.completions.create({
    model: 'mimo-pro',
    messages: [{
      role: 'system',
      content: UNIVERSE_SYSTEM_PROMPT
    }, {
      role: 'user',
      content: `创建宇宙：${keywords}`
    }],
    max_tokens: 4000,
    temperature: 0.8
  });
  return parseUniverseResponse(response);
}

// 角色对话
async function chatWithCharacter(characterId, userMessage) {
  const context = await getCharacterContext(characterId);
  return mimoClient.chat.completions.create({
    model: 'mimo-pro',
    messages: [...context, {
      role: 'user',
      content: userMessage
    }],
    max_tokens: 2000
  });
}
```

## 6. 分布式宇宙架构（Distributed Universe Architecture）

> **创新亮点**：藏子世间创新性地引入"用户贡献算力"模式，让创作者可以自建宇宙节点，其他用户可以进入体验。这种架构大幅降低平台成本，同时赋予创作者更大的创作自由度。

### 6.1 架构设计理念

#### 6.1.1 核心理念

传统UGC平台的所有内容都存储在中心化服务器上，随着用户和内容增长，成本呈指数级上升。藏子世间采用**分布式架构**，让创作者可以选择：

- **云端托管**：由平台提供算力（适合普通用户）
- **本地节点**：创作者用自己的设备运行（适合专业创作者）
- **混合模式**：核心内容在云端，重度资源在本地

#### 6.1.2 架构对比

| 维度 | 传统中心化架构 | 藏子世间分布式架构 |
|-----|--------------|-----------------|
| 存储成本 | 平台承担全部 | 用户可选择分担 |
| 算力成本 | 平台承担全部 | 创作者可贡献本地算力 |
| 扩展性 | 受服务器容量限制 | 无限扩展 |
| 创作自由度 | 受平台功能限制 | 可自定义一切 |
| 内容多样性 | 同质化严重 | 极度丰富 |

### 6.2 技术实现方案

#### 6.2.1 创作者节点计划

```
┌─────────────────────────────────────────────────────┐
│                   创作者节点架构                      │

                            │
        ┌───────────────────┼───────────────────┐
        ▼                   ▼                   ▼
┌───────────────┐   ┌───────────────┐   ┌───────────────┐
│   轻量节点      │   │   标准节点      │   │   专业节点      │
│  (云端托管)    │   │  (本地运行)    │   │  (本地+云GPU)  │
├───────────────┤   ├───────────────┤   ├───────────────┤
│ • 文本生成     │   │ • 文本+图像    │   │ • 全模态支持   │

#### 6.2.2 本地计算+云端索引模式

这是最具成本效益的架构方案：

**创作者端**：
- 在自己电脑上运行AI模型生成宇宙内容
- 使用本地存储保存所有资产（图片、音频、3D模型等）
- 只将"元数据"上传到云端：
  ```json
  {
    "universeId": "xxx",
    "name": "赛博大唐",
    "description": "2077年的长安城...",
    "tags": ["科幻", "赛博朋克", "西游记"],
    "entryPoints": ["长安城门", "菩提黑客组织"],
    "nodeAddress": "p2p://creator-node-xxx",
    "status": "online"
  }
  ```

**平台端**：
- 仅存储索引数据（每个宇宙 < 1KB）
- 提供发现、搜索、推荐功能
- 处理P2P连接和身份验证

**访问者端**：
- 通过平台发现感兴趣的内容
- 点击进入时，P2P连接到创作者节点
- 动态下载内容并本地渲染

| **节省比例** | **85%** | **92.5%** | **96%** |
