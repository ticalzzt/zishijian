## 1. Project Overview

### 1.1 Project Positioning

Zishijian (藏子世间) is an AI-powered interactive novel universe platform dedicated to creating immersive, user-participatory storytelling experiences. Built upon the foundation of the Chinese classical mythology "Journey to the West" (西游记), the platform innovatively integrates AI content generation, multi-universe parallel narratives, and user co-creation mechanisms.

### 1.2 Core Values

| Core Value | Description |
|-----------|-------------|
| **AI-Empowered Creation** | Leverage large language models to automatically generate universe settings, character dialogues, and story branches, significantly lowering the barrier to creation |
| **Multi-Universe Fusion** | Introduce the concept of "Jue Character Opportunity" (觉字机缘), where countless parallel worlds merge through karmic connections, creating infinite possibilities |
| **User Co-Creation Ecosystem** | Every user can become a universe creator, with their created content available for others to explore and interact with |
| **Immersive Experience** | Create an immersive reading experience through AI dialogue, dynamic scenes, and multi-modal content |

### 1.3 Origin of the Project Name

The name "藏子世间" carries profound meaning:

- **"藏" (Cáng)**: Derived from the "Scripture Repository" (藏经阁) in Journey to the West, symbolizing a treasure trove of knowledge and the inheritance of wisdom
- **"子" (Zǐ)**: Represents the generational hierarchy within Patriarch Subhuti's disciples, where "子" denotes entry-level disciples
- **"世间" (Shìjiān)**: Refers to the vast array of parallel universes comprising the world

Combined, "藏子世间" represents a creative and exploratory space that gathers countless stories and embraces diverse universes.

| **Web Novel Enthusiasts** | Love reading web novels, desire immersive experiences | Provides interactive reading, satisfying curiosity about story directions |

## 2. World-building & Innovation

### 2.1 The "Jue Character Opportunity" (觉字机缘) World System

Zishijian's world system is based on "Journey to the West," constructing a unique hierarchy of divine disciples:

#### 2.1.1 Patriarch Subhuti's Disciple Hierarchy

```
觉 (Jué) → 圆 (Yuán) → 悟 (Wù) → 空 (Kōng) → 我 (Wǒ) → 欲 (Yù) → 智 (Zhì) → 慧 (Huì) → 本 (Běn)
```

- **"觉" Generation**: Patriarch-level figures capable of creating universes
- **"圆" Generation**: Practitioners who have attained enlightenment, capable of perceiving and entering parallel universes
- **"悟" Generation**: Like Sun Wukong, those who have proven their worth but are still in training
- **"空" Generation and Below**: Those still in training, not yet having broken through key stages

#### 2.1.2 Protagonist Setting

Our protagonist is an ordinary person of the "圆" generation who, through fortunate circumstances, gains the ability to perceive multiple universes. As a "圆" generation member, they:
- Do not possess the ability to create universes like the "觉" generation
- Can enter and explore universes created by other practitioners
- Serve as the user's "avatar" or "projection" when entering Zishijian

#### 2.1.3 The Jue Character Opportunity

"The Jue Character Opportunity" is the core setting driving the narrative:

> Countless parallel worlds began to merge due to a certain opportunity belonging to those of the "觉" generation. Each "觉" generation being may create a unique universe, and the barriers between these universes are gradually dissolving, allowing users to freely travel between them.

### 2.2 Innovative Multi-Universe Fusion Narrative Model

Traditional web novels use linear or simple branching narratives, while Zishijian innovatively introduces **Multi-Universe Fusion Narrative**:

| Traditional Model | Zishijian Innovation |
|------------------|---------------------|
| Single-line narrative | Infinite parallel universes existing simultaneously |
| Author-led story direction | User choices determine branches, AI generates content in real-time |
| Fixed world settings | Dynamically evolving universe systems |
| One-way story consumption | Users are both readers and creators |

#### 2.2.1 Universe Types

Universe types users can explore or create include:

1. **Classic Journey to the West Universe**: Parallel worlds recreating original storylines
2. **Modern Cultivation Universe**: Journey to the West characters in brand-new stories set in modern cities
3. **Interstellar Journey to the West Universe**: Fusion of Journey to the West elements with science fiction
4. **Fully Original Universe**: Unique world settings independently created by users

### 2.3 Unique User Participatory Creation Experience

Zishijian provides an unprecedented user participation mechanism:

#### 2.3.1 Character Selection and Identity Immersion

Users can choose to enter different "universe portals," each corresponding to a unique character or event. For example:
- Enter as a bystander watching Sun Wukong's Havoc in Heaven
- Join the pilgrimage team as a junior disciple on adventures
- Explore completely original storylines as a parallel world cultivator

#### 2.3.2 Universe Co-Creation Mechanism

```
User A creates universe → User B enters to explore → User B participates in story → Universe naturally evolves
        ↓                       ↓                    ↓
   Publish universe        Comment/Interact     Branch storylines
```

Content created by users accumulates as platform assets, while other users' interactive behaviors feed back to enrich the original content, forming a positive cycle.

## 6. Distributed Universe Architecture

> **Innovation Highlight**: Zishijian innovatively introduces a "user-contributed compute" model, allowing creators to self-host universe nodes that other users can explore. This architecture significantly reduces platform costs while granting creators greater creative freedom.

### 6.1 Architecture Design Philosophy

#### 6.1.1 Core Concept

Traditional UGC platforms store all content on centralized servers, where costs grow exponentially with users and content. Zishijian adopts a **distributed architecture**, allowing creators to choose:

- **Cloud Hosting**: Platform provides compute (suitable for casual users)
- **Local Nodes**: Creators run on their own devices (suitable for professional creators)
- **Hybrid Mode**: Core content on cloud, heavy resources locally

#### 6.1.2 Architecture Comparison

| Dimension | Traditional Centralized | Zishijian Distributed |
|-----------|------------------------|----------------------|
| Storage Cost | Platform bears all | Users can share burden |
| Compute Cost | Platform bears all | Creators contribute local compute |
| Scalability | Limited by server capacity | Unlimited expansion |
| Creative Freedom | Limited by platform features | Fully customizable |
| Content Diversity | Homogenized | Extremely rich |

### 6.2 Technical Implementation

#### 6.2.1 Creator Node Program

```
┌─────────────────────────────────────────────────────┐
│               Creator Node Architecture              │
└─────────────────────────────────────────────────────┘
                            │
        ┌───────────────────┼───────────────────┐
        ▼                   ▼                   ▼
┌───────────────┐   ┌───────────────┐   ┌───────────────┐
│  Light Node   │   │ Standard Node │   │ Pro Node      │
│ (Cloud Host)  │   │ (Local Run)   │   │ (Local+GPU)   │

```

#### 6.2.2 Local Compute + Cloud Index Mode

This is the most cost-effective architecture solution:

**Creator Side**:
- Run AI models on own computer to generate universe content
- Use local storage for all assets (images, audio, 3D models, etc.)
- Only upload "metadata" to cloud:
  ```json
  {
    "universeId": "xxx",
    "name": "Cyber Tang",
    "description": "Chang'an City in 2077...",
    "tags": ["scifi", "cyberpunk", "journey-west"],
    "entryPoints": ["Chang'an Gate", "Bodhi Hacker Org"],
    "nodeAddress": "p2p://creator-node-xxx",
    "status": "online"
  }
  ```

**Platform Side**:
- Only store index data (each universe < 1KB)
- Provide discovery, search, recommendation functions
- Handle P2P connections and authentication

**Visitor Side**:
- Discover interesting content through platform
- P2P connect to creator node when entering
- Dynamically download content and render locally
